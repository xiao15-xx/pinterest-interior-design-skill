# Chrome Pinterest Workflow

## Purpose

Use this reference for Pinterest searches that depend on the user's existing Chrome login and for creating persistent candidate projects.

## Browser Sequence

1. Follow `chrome:control-chrome` bootstrap instructions exactly, including full browser documentation, session naming, exact tab claiming, and finalization.
2. Open one direct Pinterest pins search URL generated by `search_planner.py`, or use Pinterest's visible search box when direct navigation cannot be verified.
3. Check the visible page before collecting anything. Stop for login, CAPTCHA, permission, or safety prompts as required by the Chrome skill.
4. Prefer the tab's `pageAssets` capability when available:
   - list the page-provided image inventory;
   - keep only visible, relevant image assets;
   - select at most 16 assets;
   - bundle only those selected assets.
5. Do not guess tab IDs, asset IDs, URL variants, or Pinterest original-image URLs.
6. If `pageAssets` is unavailable, use visible Pin pages or user-visible downloads one at a time. Do not add a scraper fallback.

## Candidate Metadata

Before `Build`, create a UTF-8 JSON array with one object per source file:

```json
[
  {
    "FileName": "downloaded-file.jpg",
    "ImageUrl": "https://i.pinimg.com/...jpg",
    "PinUrl": "https://www.pinterest.com/pin/.../",
    "AssetId": "page-asset-id",
    "Clarity": 9,
    "FullSpace": true,
    "Realism": 9,
    "CleanComposition": 8,
    "StyleMatch": 9
  }
]
```

Use an empty string when a URL or asset ID is not visible. Never fabricate missing attribution.

## Persistence Rules

- Keep the asset bundle temporary; the `Build` action creates the durable project.
- Never edit, recompress, rename in place, or overwrite the source bundle before `Build`.
- Use stable candidate numbers from the generated project for every later discussion.
- Show the contact sheet immediately after building so the user can request changes by number.
- Treat `manifest.json` as authoritative when resuming a prior project.

## Recovery

- For extension or communication failures, read the Chrome skill's `chrome-troubleshooting` documentation before any retry.
- For file download behavior or permission prompts, read its `file-management` and `confirmations` documentation.
- If direct navigation hangs, inspect existing tabs and visible state before taking another navigation action. Do not loop through query URLs.
- Keep an unfinished Pinterest tab as a handoff only when the user must log in, solve a CAPTCHA, or make a choice on the live page.
